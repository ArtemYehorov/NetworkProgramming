Клиент-серверная архитектура\технология

Реализует сетевое взаимодействие двух узлов , в котором узлы играют роли Клиента и Сервера.

Сеть (ЛВС - локальная вычислительная сеть, LAN - Local Area Network/Network) - совокупность сетевых узлов и каналов связи между ними
Сетевой узел - активный (вычисляющий) элемент сети с назначенным сетевым именем или сетевым адрессом(ПК, маршрутезатор, принтер, сканнер, телефон...)
Канал связи - способ обмена данными между узлами.

Топология сети - схема (устрйоство) сети, организация каналов связи 
= кольцо - коммуникационный канал образует замкнутую цепь, все узлы подключаются к этому каналу
= звезда - некоторые сетевые узлы являются концетраторами , подключающими подсеть других узлов (современная топология)
= шина - похожа на кольцо, но канал не замкнут

Сетевой протокол - набор правил, по которым формируется сообщение между узлами.
 = TCP - P2P протокол, сообщение имеет адресата.
 = UPD - широковещательный протокол, данные получают все узлы.

 Сетевая технология - совокупная характеристика: топология, протокол, дргие характеристики свойственные данной сети. Современная-популярная технология  - Ethernet(TCP/IP, звезда)

 IP - сетевой адресс, число присваемое сетевому узлу.
 IPv4 - число 32 бита, разделённое для простоты на 4 байта: 127.0.0.1
 IPv6 - 128 бит, разделяемый на группы через ":"

 Порт - число,присваемое процессу (работоющей программе) на сетевом узле - способ разделения разных программ на одном узле.
 Процесс "Слушает" (Listen) порт

 EndPoint - идентификатор процесса - адресс узла + порт

 Кодировка канала - способ приобразования исходных данных в бинарные, передаваемые по каналу

 Прикладной протокол - набор правил, по которым анализируются данные, переданные по сети
 например HTTP 

 ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
 Сервер - роль сетевого узла, при которой узел принимает запрос, обрабатывает и, обычно, возвращает ответ - (Callee - тот, которого вызывают, кому звонят)
 Сервер (как процесс) постоянно активен , слушает порт в ожидании запроса от клиента 

 Клиент - роль, при которой узел инициирует "общение" , посылая серверу запрос - (Caller - звонящий вызывающий)
 Клиент (как процесс) проявляет импульстную активность : подключение-обмен данными-отключение
 
 После установки соеденения роли "сравниваются" и каждый из участников эквивалентен

 Сокет - программная абстракция, скрывающая под собой процесс обмена данными междду программами (процессами).
 С т.з. программы происходит запись в сокет и чтение из сокета, остальные детали процесса скрывабтся термином "сокет"
